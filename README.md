### ¿Qué vamos a aprender hoy?

Vamos a recorrer **el ciclo completo de un proyecto de analítica de datos**, igual que lo hace un analista profesional en una empresa real. No es teoría suelta: tomamos un problema de negocio de verdad y lo resolvemos de principio a fin.

| Etapa | ¿Qué hacemos? | ¿Para qué sirve en la vida real? |
|---|---|---|
| 1. Entender el negocio | Leer el reto: ¿qué nos están preguntando? | Sin pregunta clara, los datos no sirven |
| 2. Cargar datos (bases de datos) | Traer la información a Python | Las empresas guardan TODO en datos |
| 3. Limpieza | Arreglar errores, espacios, formatos | El 80% del trabajo real es limpiar |
| 4. EDA (exploración) | Mirar, graficar, encontrar patrones | Aquí nacen los *insights* |
| 5. RFM | Segmentar clientes por su comportamiento | Saber a QUIÉN hablarle y CÓMO |
| 6. Modelo predictivo | Anticipar quién dejará de venir | Actuar ANTES de perder al cliente |
| 7. Decisión | Convertir números en acciones | El objetivo final del analista |

> **Idea central del día:** Un dato por sí solo no vale nada. El valor aparece cuando lo convertimos en una **decisión**.
